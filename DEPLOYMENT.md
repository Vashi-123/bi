# Инструкция по деплою на Linux (Ubuntu/Debian)

Эта инструкция поможет вам запустить дашборд в фоновом режиме и настроить автоматическое обновление.

## 1. Подготовка сервера

Установите необходимые системные компоненты:

```bash
# Обновление системы
sudo apt update && sudo apt upgrade -y

# Установка Node.js (V20+)
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# Установка Python и Pip
sudo apt install -y python3 python3-pip

# Установка PM2
sudo npm install -g pm2
```

## 2. Установка приложения

Перенесите файлы проекта на сервер и выполните установку зависимостей:

```bash
# Установка зависимостей бэкенда
pip install -r backend/requirements.txt

# Установка и сборка фронтенда
cd frontend
npm install
npm run build
cd ..

# Делаем скрипт обновления исполняемым
chmod +x update.sh
```

## 3. Запуск через PM2

Запустите оба сервиса одной командой:

```bash
pm2 start ecosystem.config.js
```

Чтобы сервисы запускались автоматически при перезагрузке сервера:
```bash
pm2 save
pm2 startup
# (Скопируйте и выполните команду, которую выдаст PM2)
```

## 4. Обновление кода и данных

### Обновление дизайна / функционала
Если вы изменили файлы кода (например, `page.tsx`), просто запустите скрипт обновления:
```bash
./update.sh
```

### Обновление данных (Parquet)
Просто закидывайте новые `.parquet` файлы в папку:
`./project_data/processed/final_df/`

**Дашборд увидит их автоматически.** Из-за кэширования изменения могут появиться с задержкой до 5 минут. Если данные нужны мгновенно, выполните:
```bash
pm2 reload powerbi-backend
```

## 5. Полезные команды PM2

- `pm2 status` — проверить, работают ли сервера.
- `pm2 logs` — посмотреть ошибки и логи.
- `pm2 stop all` — остановить всё.
- `pm2 restart all` — перезапустить всё.
